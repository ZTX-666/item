# datefinder 1.0.0rc2

Release date: 2026-03-23

## Highlights

- Keeps the v2 compatibility engine as the default for `find_dates(...)`.
- Ships the next issue-triage wave of compatibility behaviors and flags.
- Expands regression coverage for ambiguity, multiline extraction, and opt-in parsing modes.

## New Compatibility Controls

- `allow_month_only=True` (default): enables month-only parsing such as `"in May"` -> `YYYY-05-01`.
- `allow_compact_numeric=False` (default): keeps compact numeric parsing disabled unless explicitly enabled.
- `allow_multiline=True` (default): allows cross-line extraction; can be disabled for strict line-by-line parsing.

CLI equivalents:

- `--no-month-only`
- `--compact-numeric`
- `--no-multiline`

## Fixes Included in rc2

- Avoids false positives from dotted time ranges where `month + day` could be inferred incorrectly (e.g. `April 09.00`).
- Includes previously merged issue fixes that were not available in the earlier published RC build.

## Validation Summary

- Local test suite: `92 passed`.
- Rust kernel + Python package validated in editable install with CLI and API regression coverage.

## Portability Notes

- Wheels remain platform-specific (Linux manylinux/musllinux, macOS x86_64/arm64, Windows x86_64).
- If no wheel matches, installation falls back to source build and requires Rust tooling.
