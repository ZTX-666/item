# datefinder 1.0.0

Release date: 2026-03-25

## Highlights

- Promotes the `1.0.0rc3` candidate line to final `1.0.0`.
- Keeps parser/API behavior validated during RC cycles.
- Ships the Rust-kernel-backed v2 compatibility runtime as default behavior.

## Runtime and API

- `find_dates(...)` defaults to the v2 compatibility engine.
- Legacy behavior remains available via:
  - `find_dates_legacy(...)`
  - `find_dates(..., engine="legacy")`
- Typed extraction API remains available via `extract(...)` for absolute, relative, and duration values.

## Packaging and Distribution

- Final PyPI release includes wheels for:
  - manylinux2014 (`x86_64`, `aarch64`)
  - musllinux_1_2 (`x86_64`, `aarch64`)
  - macOS (`x86_64`, `arm64`)
  - Windows (`x86_64`)
- Source distribution remains available and builds from source when no matching wheel is present.

## Notes

- This GA release keeps the compatibility flags introduced in RCs:
  - `allow_month_only` (default `True`)
  - `allow_compact_numeric` (default `False`)
  - `allow_multiline` (default `True`)
- Ambiguous shorthand such as `9.6 20:30` remains intentionally unsupported by default to limit false positives.
