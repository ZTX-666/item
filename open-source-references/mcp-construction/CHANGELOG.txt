COMMIT 4: 60883a0 — Fix 6 critical security vulnerabilities
========================================================================

C1: Passwordless login bypass (src/auth/routes.ts)
  - Changed: if (user.passwordHash) → if (!user.passwordHash) return 403
  - Always validates password now

C2: OAuth client secret never validated (src/auth/oauth.ts)
  - Added bcrypt validation of client_secret in exchangeAuthorizationCode()
  - Added bcrypt validation of client_secret in refreshAccessToken()
  - Updated routes.ts to pass clientSecret through

C3: XSS in OAuth consent page (src/auth/routes.ts)
  - Added escapeHtml() function
  - Applied to all 8 interpolated values in consent page HTML

C4: Trial expiration never enforced (src/billing/middleware.ts)
  - Restructured: check trial expiration BEFORE ACTIVE_STATUSES pass-through
  - 'trialing' was in ACTIVE_STATUSES, making expiry check unreachable

C5: Private key world-readable
  - chmod 600 on keys/private.pem, keys/public.pem, .env

C6: SSH password authentication enabled
  - Overwrote /etc/ssh/sshd_config.d/50-cloud-init.conf
  - Set PasswordAuthentication no (first-match-wins in OpenSSH)

*** IMPORTANT: Admin scope change ***
Currently, every agent that signs up gets 'admin' scope by default
(src/auth/signup.ts:125). This grants access to:
  - approve_change_order (financial impact)
  - create_user / update_user_role / remove_user (user management)
  - update_org_settings (org configuration)
  - import_cost_data (bulk data import)
  - /admin/* HTTP endpoints (usage analytics, billing, subscription changes)
This is fine for single-user orgs but risky for multi-user orgs
connecting third-party agents. Should remove 'admin' from the default
signup scope before marketing to larger teams.
