# 测试：公司 HiAgent 能否访问「本地数据库」

## 先弄清两件事

| 问题 | 答案 |
|------|------|
| 赤瞳的 `wacli.db` 能否在 HiAgent「创建数据库」里直接填？ | **不能**。那是 **SQLite 文件**，表单要的是 **MySQL + host:端口**。 |
| 公司云端能否访问你电脑上的库？ | **默认不能**。需要公网/专线/VPN + 本机起 MySQL 并暴露端口（公司规定可能禁止）。 |

本目录脚本帮你 **分步测清**，避免盲目填表。

---

## 测试顺序（建议按序做）

### 步骤 0：本机数据是否存在

```powershell
cd source\hiagent-local-test
powershell -ExecutionPolicy Bypass -File .\test-db-reachability.ps1
```

看输出里 `wacli.db` 是否找到、消息条数。

---

### 步骤 1：测「公司库 ← 你本机」（通常失败，但可验证）

HiAgent 填的是 **你的公网 IP + 3306** 时，公司要从外网连进来。

脚本会检查：

- 本机是否监听 3306（MySQL）
- 公网 IP 是什么
- 本机防火墙是否放行 3306

**多数家庭/公司办公网：3306 不会被公网打通** → 连接测试会失败，这是正常现象。

---

### 步骤 2：测「你本机 → 公司 MySQL」（常常成功）

在 HiAgent 里创建好的 **公司 MySQL**，从你电脑用脚本连一次：

```powershell
$env:MYSQL_TEST_HOST = "公司给的host"
$env:MYSQL_TEST_PORT = "3306"
$env:MYSQL_TEST_USER = "用户名"
$env:MYSQL_TEST_PASS = "密码"
$env:MYSQL_TEST_DB   = "库名"
python test_company_mysql.py
```

- 若 **成功**：说明你可以把数据 **同步上去**，HiAgent 再连同一台公司库即可（不是直连 SQLite）。
- 若 **失败**：问 IT 是否要 VPN、白名单、或只能用内网地址。

---

### 步骤 3：HiAgent 控制台「连接测试」（最终依据）

在平台「创建数据库」里填 IT 提供的 **公网 host**（截图要求），点 **连接测试**。

- **通过**：平台能连该 MySQL；是否已有 WhatsApp 数据取决于你是否已同步。
- **失败**：把脚本里「步骤 1/2」结果发给 IT，不要反复用 `127.0.0.1` 测试。

---

### 步骤 4（仅个人实验、公司可能禁止）：隧道暴露本机 MySQL

仅用于 **验证「理论上云端能否连到你这台机器」**，不要在办公网/真实数据上使用。

```text
本机 Docker MySQL (3306) + cloudflared tcp ...
```

详见脚本输出中的 `[实验]` 说明。与 publish12 公网隧道一样，可能违反公司（2）（3）条。

---

## 结论对照表

| 测试结果 | 含义 | 建议 |
|----------|------|------|
| 步骤 1 失败、步骤 2 成功 | 公司访问不了你本机，但你能访问公司库 | **同步数据到公司 MySQL**，不要暴露本机 |
| 步骤 2、3 都成功 | 平台与公司库连通正常 | 做增量同步 + 数据集配置 |
| 步骤 0 无 wacli.db | 本地尚无数据 | 先赤瞳 sync |
| 只有步骤 0 成功 | 仅本机有数据 | 用 `export_sample_json.py` 或 MCP 内网方案 |

---

## 合规提醒

- 不要用真实商务 WhatsApp 数据做公网暴露实验。
- 办公环境部署本地 MySQL/隧道前需 IT 与安全审批。
