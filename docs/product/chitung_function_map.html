Akame Safety Intelligence Platform

赤瞳平台五大功能板块

平台按“视觉安全、消息协同、文档生成、知识结构化、中台调度”拆分职责。 四个业务系统负责具体场景，赤瞳中台负责统一大模型 API、多智能体编排、飞书/Chatbot/技能加载和跨系统打通。

架构原则 业务闭环必须人工确认 所有 LLM 调用统一走中台 文件、证据、消息均可追溯

赤瞳中台

统一 LLM Gateway、多智能体协作、FastAPI 聚合接口、飞书机器人与 Chatbot 入口、技能加载、运行状态监控、桌面服务托管。 它不是替代四个业务系统，而是作为“调度层与信任层”把数据、工具、人工确认和输出文件串起来。

视觉安全闭环

赤瞳守护者

VLM 视觉识别、RTMP 巡检、隐患候选、台账和整改闭环。

核心功能

  • RTMP 摄像头截图、选择已配置摄像头运行巡检。
  • VLM 检测 PPE、机械、临边、高风险目标。
  • 生成隐患候选，人工确认后写入 safety_cases。
  • 隐患台账查询、筛选、状态操作。
  • 整改通知、分包商确认、复查、关闭。

已接入输出

  • 巡检截图、VLM 结果、证据文件可打开。
  • 最近生成文件历史记录证据路径。
  • 整改报告可从台账生成。

归属边界

  • 负责“看见风险、形成案件、推动闭环”。
  • 不直接负责大模型配置和跨系统编排。
消息与协同

赤瞳灵讯

WhatsApp 消息收集、分析、发送确认，以及 SQLite 消息数据库。

核心功能

  • WhatsApp 归档服务健康检查与连接器配置。
  • 消息搜索、附件下载、施工群信息沉淀。
  • 消息入库到 SQLite，形成 source_messages。
  • 从消息文本提取区域、分包商、隐患线索。
  • 通知草稿生成、人工确认、发送结果回写。

当前状态

  • WhatsApp 搜索/下载依赖本地 archive app-server。
  • 发送底层目前是 confirmed_pending_sender，不伪装成真实已发送。
  • 飞书 webhook 已可真实发送并回写状态。

归属边界

  • 负责“消息收集、消息数据库、群组协同”。
  • 隐患判断结果交给守护者和中台编排。
人机协同文档

闪闪文档

文档生成、表格、信件、Diff 审阅、人工采纳。

核心功能

  • AI 文档改写预览，红绿 Diff 展示 +N/-N。
  • 表格模板搜索、字段预填、DOCX 草稿生成。
  • 人工采纳后才写入表单记录。
  • 整改通知、信件、日报、报告草稿生成。
  • DOCX、payload、Markdown 输出可打开定位。

文档类型

  • 安全表格：T006 等模板。
  • 整改信件：正式通知、分包确认、关闭意见。
  • 报告：安全日报、整改报告、社区报告 DOCX。

归属边界

  • 负责“写、改、审、采纳”。
  • OCR 和知识库更新归耀耀慧读，中台只负责调度。
知识结构化

耀耀慧读

OCR、文档结构化、舆情天气抓取、安全文档更新与内部 RAG。

核心功能

  • OCR 识别 PDF、图片、扫描文档。
  • 文档表格抽取、字段结构化、政策条款切片。
  • 天气、酷热、新闻、安全公告外部风险抓取。
  • 安全制度与表格模板库更新。
  • 内部 RAG 数据库与知识查询。

已接入基础

  • 香港天气与安全新闻风险简报入口。
  • 安全政策模板库与表格索引。
  • 文档结构化/OCR 工具预留在工具层。

归属边界

  • 负责“读懂资料、更新知识、提供检索”。
  • 生成正式文件交给闪闪文档，人机工作流交给中台编排。

典型协作链路

1. 消息进入

赤瞳灵讯收集 WhatsApp / 飞书线索，沉淀到 SQLite。

2. 知识增强

耀耀慧读补充 OCR、天气、新闻、制度条款和 RAG 上下文。

3. 视觉核验

赤瞳守护者通过 RTMP/VLM 形成隐患候选和证据文件。

4. 文档生成

闪闪文档生成表格、整改通知和报告，人工 Diff 采纳。

5. 中台闭环

赤瞳中台统一调用 LLM、工具、飞书/Chatbot，并回写状态。
