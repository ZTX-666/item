Skill 相关能力

把业务专家经验沉淀成可配置的技能说明和工具依赖。

  • 技能目录覆盖隐患 intake、视觉巡检、知识问答、文档编辑、外部风险监听。
  • 前端可查看启停状态、依赖健康检查和最近运行记录。
  • 替代临时口头经验传递，让不同场景调用同一套标准化能力。

Workflow 相关能力

把多步作业变成可追踪的自动流程。

  • 模板化定义“天气抓取、风险持久化、简报生成、发送确认”等步骤。
  • 每一步记录运行状态、工具结果和审计事件，便于复盘。
  • 替代人工跨系统复制粘贴和反复催办。

自动化相关能力

让固定频率和高风险触发任务自动产生草稿。

  • 外部讯息 Scheduler 可定时拉取天气、监管、安全新闻。
  • 视觉巡检支持批量摄像头检测并输出候选隐患和图片证据。
  • 所有关键动作先进入待确认，保证自动化可控。

Agent 相关能力

让 AI 负责理解意图、选择工具、生成解释和下一步建议。

  • LLM Gateway 统一脱敏、压缩上下文和 JSON 格式输出。
  • Agent Trace 在前端展示计划、工具调用和结果阶段。
  • 替代人工判断“该找哪个系统、填哪个表、发给谁”。

落地流程：从现场问题到闭环执行

1. 多入口采集 桌面工作台、飞书机器人、WhatsApp 群、摄像头、文档上传。
2. 意图识别 规则优先识别业务场景，复杂表达交给 LLM 结构化判断。
3. 知识增强 结合制度文档、历史台账、外部风险和群消息上下文。
4. 工具执行 AgentToolbox 调用 OCR、VLM、DOCX、飞书、WhatsApp、SQLite。
5. 人工确认 外发、写文件、创建和关闭隐患前统一生成确认卡片。
6. 审计闭环 Job、Workflow Run、Audit Event 保留全过程证据。

2 分钟技术讲解词

解决什么痛点

  • 过去安全信息散落在摄像头、群消息、文档表格和人工台账中，靠人盯屏、翻群、填表、催办。
  • 赤瞳把这些入口统一接入中台，让 AI 先整理成候选结论，再交给人确认。

技术怎么运作

  • 中台先识别意图，再调 Skill/Workflow，按需调用 VLM、OCR、RAG、LLM、DOCX、通讯工具。
  • AI 不直接替人做最终决策，而是生成草稿、卡片、Diff 和风险建议。

创新与落地性

  • 系统不是单点模型演示，而是连接现有群聊、飞书、本地数据库和文档模板的业务闭环。
  • 人审确认和审计记录保证现场可控，适合项目部逐步上线。

代码实现对应关系

中台与 API

chitung-center 暴露健康检查、聊天、技能、工作流、视觉巡检、文档、RAG、确认和机器人事件接口。

  • 核心文件：app.py、orchestrator.py、intent_router.py、workflow_engine.py。

桌面工作台

chitung-frontend 使用 Vue Router 分为 guardian、docmate、lingxun、center、yaoyao、automation 等页面域。

  • 核心页面：VisualPatrolPage、WhatsAppOpsPage、AIAssistantPage、AutomationPage。
  • 统一 API 客户端：chitungApi.ts。

工具执行层

agent-toolbox 封装本地工具，提供 HTTP Tools API 和 MCP Adapter。

  • 工具类别：VLM、OCR、RTMP、DOCX、飞书、WhatsApp、表格、报表、SQLite。
  • 中台通过 toolbox_client 统一调用，保留工具审计。
