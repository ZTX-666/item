# Visual Patrol

从 CCTV、RTMP 或本地图片抽帧，执行 YOLO/VLM 安全巡检，并把证据写入本地 SQLite。

## Rules

- **地盘 / 全量巡检**：当用户提到「地盘」「全部摄像头」「N 支摄像头」（如 11 支）、或未指定单路名称时，必须对**所有已启用摄像头**逐路抽帧检测，不得默认只检 1 路。
- **单路巡检**：仅当用户明确点名某一路（如「斜坡03」「施工區域01」）时，才只检测该路。
- 优先使用真实摄像头或用户选择的视频源抽帧，不得静默回退到本地兜底图片。
- 抽帧失败时必须返回明确错误、摄像头编号、失败原因和可重试建议；并在汇总中列出**已成功 / 失败 / 未检**路数。
- YOLO 标注图、原始抽帧图、检测 JSON、VLM 分析结果和用户检测要求都要保留路径或结构化字段。
- 视频巡检结果应展示标注图片、目标数量、风险等级、建议动作和 SQLite 存储位置。
- 汇总文案必须写清：`已完成 X/Y 路摄像头检测`；若 X < Y，需提示剩余路数待补检。

## Preferred Tools

- `workbench_video_detection`
- `rtmp_snapshot`
- `vlm_detect`
- `ingest_vlm_hazards`

## Workflow

1. 解析用户意图：全地盘巡检 vs 单路点名
2. 解析检测目标（如吸烟、PPE、机械半径、吊运）
3. 对范围内每路摄像头调用 `workbench_video_detection`
4. 汇总各路证据、失败路数与建议动作

## Example Intents

| 用户说法 | 摄像头范围 |
|---------|-----------|
| 检查地盘 11 支摄像头是否有人吸烟 | 全部已启用（11 路） |
| 检测地盘吊运是否符合安全要求 | 全部已启用（11 路） |
| 检查斜坡03 有没有未戴安全帽 | 仅「斜坡03」 |
| 全盘巡检 PPE | 全部已启用 |
| 检查摄像头 | 全部已启用（视觉巡检默认） |

## Frontend

- 展示视频巡检结果卡片 `video_detection_report`。
- 多路巡检时按摄像头分组展示 `snapshot_url` / `annotated_url`，避免重复堆叠同一路图片。
- 如果存在 `snapshot_url` 或 `annotated_url`，优先展示图片证据。
- 如果工具返回失败，直接显示失败原因，不要显示静态占位图冒充检测结果。

## Boundaries

- 不得在未完成全量巡检时声称「已完成地盘检查」
- 不得把单路结果冒充全盘结论
