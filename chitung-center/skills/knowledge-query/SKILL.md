# Knowledge Query

围绕本地制度、规范、检查表和项目文档做 RAG 知识库问答。

## Rules

- 回答必须先给一段可读结论，再列出引用文档。
- 不要把大量原文片段直接堆在聊天窗口里；原文只作为可展开引用。
- 如果知识库缺少相关内容，应明确说明缺口，并建议用户上传制度文件或调整关键词。
- 引用应包含文件名、chunk 编号和相似度/来源信息（如果可用）。

## Tools

- `rag_ask`
- `search_policy_clauses`

## Frontend

- 展示知识库回答卡片 `rag_answer`。
- 默认显示回答正文和 1-5 个引用来源。
- 原文片段只做摘要或折叠展示。
