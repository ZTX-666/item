本文档用于将当前项目整体复制到另一台机器或另一套 Cursor 环境时快速接手，避免遗漏今天对话中的关键决策、已完成改动和交付方式。

## 1. 项目定位

DocMate（闪闪文档）是一款 Windows 桌面级便携文稿修改助手，核心定位是：

- 人机协同：AI 负责理解指令、定位文本、生成修改建议；用户负责审阅、采纳或拒绝。
- 可控改稿：所有 AI 修改必须先展示 Diff，不能直接覆盖正文。
- 自我进化：通过用户偏好、知识库和交互记录逐步贴合个人/组织写作习惯。
- 本地便携：发布包可直接运行，数据尽量保存在本地工作区或便携目录。

关键约束仍然是：

- 粘性选区不能破坏。
- Diff 预览必须保留。
- API Key 不应暴露给前端。
- 功能性代码变更后需要执行 release。

- 斜体：使用 `StarterKit` 内置 `italic`。

- AI 生成和 Diff 审阅时格式按钮会禁用。

业务边界：

- AI 改稿读取纯文本，不会把 B/I/U HTML 传给模型。
- 如果用户先给一段文字加格式，再让 AI 替换这段文字，采纳后局部格式可能丢失。
- 该潜在冲突已确认可接受，B/I/U 定位为“人工定稿后的轻量排版工具”。

## 7. 已知风险与注意事项

### 7.1 依赖审计风险

安装 `marked` 和 `docx` 后，`npm install` 输出了依赖审计风险：

- 16 high
- 3 critical

未执行：

```bash
npm audit fix --force
```

原因：强制修复可能破坏 Electron、Tiptap、Transformers 的当前稳定依赖组合。后续如果要处理安全审计，建议单独开任务验证。

### 7.2 Tiptap 版本

不要直接升级到 Tiptap 3。

原因：

- 当前项目是 Tiptap 2 架构。
- 粘性选区、Diff 预览、自定义 mark、BubbleMenu 和 AI 替换链路都依赖现有行为。
- 为了 B/I/U 没有必要升级大版本。

### 7.3 导出 Word/PDF 是轻量首版

当前导出 Word 是将 HTML 转为纯文本段落，不完整保留复杂样式。

当前导出 PDF 会使用 Electron 临时窗口渲染 HTML 后打印。

后续如果要做正式交付级 Word/PDF，可以单独增强：

- 标题样式映射
- 加粗/斜体/下划线保留
- 表格保留
- 页眉页脚
- 公文格式模板

### 7.4 publish100 和集成 exe 的区别

`publish100/` 是精简运行包，依赖在目录内安装，可用 `DocMate.vbs` 启动。

`release/DocMate闪闪文档 4.1.37.exe` 是 electron-builder 生成的便携 exe，更适合作为桌面级便携办公软件交付。

### 7.5 语音和包体

本次没有做：

- Whisper tiny 升级 base。
- 本地向量检索。
- 大幅 node_modules 瘦身。
- 多标签页。
- 知识库加密。
- 多模型路由。

这些都建议单独立项，不要和当前已稳定的 publish100 混在一起。
