## 可优先结合的制度

1. 安全施工计划与月度自检制度
   - 相关模板：`T001` 至 `T007`
   - 可结合现有平台的巡检、VLM 识别、WhatsApp 现场反馈和飞书提醒，形成“季度计划 -> 月度自检 -> 问题整改 -> 下月关注事项”的闭环。

2. 安全事件、近危事件、停工整改命令制度
   - 相关模板：`T014` 至 `T019`，以及 `T124` 至 `T126`
   - 可把现场文字、图片、视频截图、VLM 识别结果自动整理为事件分级、整改命令草稿和通报材料。

3. 吊运、起重机、货车式起重机、物料吊重机安全制度
   - 相关模板：`T032` 至 `T061`
   - 与摄像头识别、设备证书 OCR、吊运信号员/合资格人员台账结合，适合做高风险作业前检查和每日吊运合规助手。

4. 危险品、充电、防火、工友休息室制度
   - 相关模板：`T062` 至 `T065`
   - 可结合现场巡检照片、危险品清单、消防/通道/充电点检查，生成每周检查记录和异常提醒。

5. 电力安全与配电箱检查制度
   - 相关模板：`T066` 至 `T074`
   - 适合做“配电箱入场资料审查 + 现场检查表 + 双重绝缘/接地检查”填表和证据归档工具。

6. 棚架、临时支架、密闭空间等高危工序制度
   - 相关模板：`T088` 至 `T107`
   - 可结合高危工序计划、人员委任、开工前交底、每日照片记录和 VLM 风险识别。

7. 分区责任、分判商安全表现、行为安全之星制度
   - 相关模板：`T111`、`T116` 至 `T136`
   - 适合做分区评分、分判商安全打理人评审、奖励签收和行为安全激励台账。

8. 恶劣天气、应急物资、热应激和高龄工友健康管理制度
   - 相关模板：`T148` 至 `T159`
   - 可结合天气 API、现场打卡、健康声明、饮水点/遮阳设施检查，形成“暑热风险提醒 + 工友健康检查 + 应急物资台账”。

## 建议扩充的 Agent Tools

### 1. `policy_template_search`

输入制度关键词、场景或表格名称，返回最匹配的制度章节、表格模板 ID、模板路径和填写说明。

典型用途：安全主任问“吊运前要填什么表”，agent 返回 `T037`、`T043`、`T048` 等候选模板，并解释适用条件。

### 2. `safety_form_fill`

根据模板 ID、项目资料、人员资料、现场检查结果和图片证据，自动生成已预填的 Word 表格。

第一阶段建议支持：`T002` 月度安全施工计划、`T004` 自检报告、`T006` 整改建议、`T043` 起重机检查、`T048` 货车式起重机检查、`T065` 工友休息室每周检查、`T073` 配电箱检查、`T118` 分区检查评分、`T155` 健康声明及检查。

### 3. `risk_workflow_classifier`

根据输入的作业描述、时间、地点、机械、人员和图片，判断属于特别高危、一般高危或常规工序，并输出需要准备的文件清单、提前通知时限和监管动作。

可用制度依据：正文中高危工序管理要求，例如特别高危工序开工前 1 个月、一般高危工序开工前 2 星期准备施工方案、风险评估和安全操作程序。

### 4. `stop_work_order_draft`

根据现场隐患、严重程度、责任分区、分判商、照片和整改要求，生成局部/全面/重大事件停工整改命令草稿。

相关模板：`T015`、`T016`、`T017`。输出应保留“需人工确认后发送”的状态，避免自动发出正式停工命令。

### 5. `monthly_safety_plan_assistant`

把巡检记录、VLM 识别结果、WhatsApp 现场消息、整改台账和下月施工计划汇总为月度安全施工计划和自检报告。

相关模板：`T001` 至 `T007`。这会成为平台的核心闭环工具之一。

### 6. `lifting_operation_checker`

围绕吊运安全守则，检查吊运信号员委任、起重机/打桩机入场审批、货车式起重机检查、吊具检查和物料吊重机使用安排。

相关模板：`T032` 至 `T061`。可接入 OCR 读取证书有效期，也可接入摄像头识别吊运区域人员闯入、未戴安全帽、吊物下方站人等风险。

### 7. `electrical_box_precheck`

从配电箱照片、设备资料和检查项中生成入场前安全检查表，并标记双重绝缘、接地、保护装置、标签、门锁、漏电保护等缺项。

相关模板：`T073`、`T074`。

### 8. `heat_stress_health_check`

结合天气数据、现场温度、工友年龄、健康声明、开工前身体状态检查，生成热应激风险提醒和健康检查记录。

相关模板：`T150` 至 `T159`。适合通过飞书/中海通机器人每天早上推送。

### 9. `contractor_safety_score`

把分区检查、分判商打理人评审、行为安全表彰卡、违规扣分和奖励签收统一汇总，形成分判商安全评分和月度趋势。

相关模板：`T111`、`T118`、`T123`、`T129` 至 `T136`。

### 10. `emergency_materials_register`

建立应急物资台账，定期核对数量、有效期、位置和缺口，并按恶劣天气预案生成补充建议。

相关模板：`T148`、`T149`。

## 填表 Tool 的模板资产

已保存的模板资产：

- `tables-docx`: 每个表格单独保存为 Word 模板，适合后续做“保持原格式填表”。
- `tables-json`: 每个表格的行列、单元格文字、上下文和路径，适合做字段识别、模板检索和自动填充。
- `tables-md`: 每个表格的 Markdown 预览，适合人工快速查看。
- `TABLE_INDEX.md`: 159 个表格的总索引。
- `table_index.json`: 159 个表格的机器可读总索引。
- `text/full_text.txt`: 制度正文全文，适合后续做 RAG 检索。
- `text/keyword_snippets.json`: 事故、隐患、整改、检查、高危工序等关键词片段。

## 后续落地建议

第一期先做三个最有价值的工具：`policy_template_search`、`safety_form_fill`、`monthly_safety_plan_assistant`。它们能直接把这份制度汇编转化为可问、可查、可填、可归档的工作流。

第二期再接入高风险场景：`stop_work_order_draft`、`lifting_operation_checker`、`electrical_box_precheck`、`heat_stress_health_check`。这些工具可以与 VLM、OCR、WhatsApp/飞书/中海通机器人形成强联动。
