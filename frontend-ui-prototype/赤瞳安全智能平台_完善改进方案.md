# 赤瞳安全智能平台 — 完善改进方案

> 本文档基于《赤瞳安全智能平台_产品技术方案》初稿，从产品经理视角提出改进意见，覆盖功能实现路径、操作逻辑、UI 界面定义和技术规范，使程序员可据此直接开发。

## 一、功能实现路径改进

### 1.1 问题：五阶段开发线太长，竞赛前无法全部交付

原方案分五阶段，按顺序推进。但竞赛评审只需要看到**核心闭环能跑通**，不需要全部功能上线。

**改进：改为"一个闭环 + 三个亮点"的交付策略**

```
核心闭环（MUST HAVE，2 周内完成）：
  聊天输入隐患 → AI 分类入库 → 生成整改通知草稿 → 人工确认 → 发送

亮点 A（视觉巡检，竞赛演示用）：
  CCTV 截图 → VLM 检测 → 隐患入库 → 走核心闭环

亮点 B（智能填表，竞赛演示用）：
  对话框说"帮我填 T006" → 模板检索 → 字段自动填充 → 生成 Word

亮点 C（外部风险雷达，竞赛演示用）：
  天文台预警 → 自动推送检查建议 → 关联表格模板
```

**每个亮点独立可演示，互不依赖。** 程序员可以并行开发。

## 二、软件操作逻辑

### 2.1 核心交互模型：对话框驱动一切

**设计原则：安全主任不需要学多个系统，一个对话框搞定所有事。**

系统只有一个主入口——对话框。用户用自然语言说需求，系统识别意图后调用对应工具链。

**对话框的三种响应模式：**

| 模式 | 触发条件 | 展示方式 | 示例 |
|------|---------|---------|------|
| 直接回答 | 查询类问题 | 消息气泡 | "B2 区目前有 3 条未闭环隐患" |
| 操作确认 | 执行类操作 | 确认卡片 | "即将向「地盘安全群」发送整改通知，确认？" |
| 结果展示 | 生成类操作 | 结果卡片 + 下载 | "已生成 T006 表格，点击下载 Word" |

### 2.2 核心用户流程

#### 流程 1：录入隐患（最常用，每天 10+ 次）

```

   ┌─────────────────────────────────────┐
   │  隐患识别结果                         │
   │                                     │
   │  类型：临边洞口防护缺失               │
   │  风险等级：高                         │

   │  建议模板：T006, T015, T118          │
   │                                     │
   │  [确认入库]  [修改]  [忽略]          │
   └─────────────────────────────────────┘

2. 点击"确认入库"                 → 写入 safety_cases
                                  → 自动生成通知草稿：

   ┌─────────────────────────────────────┐
   │  通知草稿                             │
```

### 2.3 操作逻辑的补充规则

**规则 1：所有执行类操作必须经人工确认**

以下操作需要弹出确认卡片，不允许静默执行：
- 发送 WhatsApp/飞书/中海通消息
- 生成 Word 文档并保存到磁盘
- 写入 safety_cases 的新记录（AI 自动入库的 VLM 结果也需确认）
- 生成警告信/罚款信/停工令

**规则 2：查询类操作直接回答，不需要确认**

- "B2 区有多少未闭环隐患？"
- "T006 表格模板在哪？"
- "今天的天气预警是什么？"

**规则 3：AI 分类结果允许一键修正**

每次 AI 分类结果卡片必须有"修改"按钮，修正后写入 `classification_feedback` 表，供后续 LLM 微调和规则优化。

### 3.2 各模块页面定义

#### 模块 1：对话（默认首页）

- **布局**：聊天消息流 + 底部输入框
- **消息类型**：

| 类型 | 渲染方式 | 示例 |
|------|---------|------|
| 文本消息 | 气泡 | "B2 区有 3 条未闭环隐患" |
| 隐患确认卡片 | 卡片，含操作按钮 | 见流程 1 |
| 通知草稿卡片 | 卡片，含发送/编辑/忽略 | 见流程 1 |

| 表格填充卡片 | 表单 + 自动填充 | 见流程 3 |

- **输入框**：
  - 支持文本输入
  - 支持粘贴图片（自动触发 OCR + 分类）
  - 支持拖拽文件（PDF/Word/Excel → 自动解析）
  - 支持语音输入（调用系统语音识别 → 文本）

### 3.3 卡片组件规范

所有对话框内的交互卡片统一遵循以下规范：


### 4.4 LLM Gateway 实现要点

```python
class LLMGateway:
    """
    统一 LLM 调用网关
    - 脱敏：发送前移除身份证、电话、完整地址
    - 最小上下文：只发送当前任务需要的字段
    - 审计：每次调用记录到 tool_audit_logs
    - JSON 校验：要求 LLM 返回结构化 JSON，校验后使用
    """

    async def call(self, task: str, context: dict, output_schema: dict) -> dict:
        # 1. 脱敏
        safe_context = desensitize(context)

        # 2. 构建 prompt（system + task + output format）
        messages = build_prompt(task, safe_context, output_schema)

        # 3. 调用 LLM
        response = await self.client.chat(messages)

        # 4. 审计
        await self.audit_log(task, safe_context, response)

        # 5. JSON 校验
        return validate_json(response, output_schema)
```

```python
    conn = sqlite3.connect(db_path)
    current = conn.execute("PRAGMA user_version").fetchone()[0]
    for version, sql in MIGRATIONS.items():
        if version > current:
            conn.executescript(sql)
            conn.execute(f"PRAGMA user_version = {version}")
    conn.close()
```

### 4.6 Skill 加载器实现

```python
class SkillLoader:
    """
    加载 SKILL.md 文件，解析为工具调用链
    - Skill 不执行代码，只定义流程
    - Skill 内引用的 tool 必须是已注册的内置 tool
    """

    def __init__(self, skills_dir: str):
        self.skills = {}
        self.index = self._build_index(skills_dir)

    def _build_index(self, skills_dir: str) -> dict:
        index = {}
        for skill_dir in Path(skills_dir).iterdir():
            skill_file = skill_dir / "SKILL.md"
            if skill_file.exists():
                meta, steps = parse_skill_md(skill_file)
                index[meta["name"]] = {
                    "description": meta["description"],
                    "steps": steps,       # [{"tool": "xxx", "params": {...}}, ...]
                    "file_path": str(skill_file)
                }
        return index

    def get_skill(self, name: str) -> dict:
        return self.index.get(name)

    def list_skills(self) -> list:
        return [{"name": k, **v} for k, v in self.index.items()]
```
