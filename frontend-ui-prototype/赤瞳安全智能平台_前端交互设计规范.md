# 赤瞳安全智能平台 — 前端交互设计规范

> 本文档解决核心问题：**一个软件既要当传统管理工具用（表格、看板、台账），又要当 AI Agent 用（对话、自动填表、自动发送），界面怎么设计才不割裂？**

---

## 一、市面上同类产品的三种解法

### 1.1 三种主流模式

| 模式 | 代表产品 | 核心做法 | 适合谁 | 优劣 |
|------|---------|---------|--------|------|
| **全沉浸式** | Manus, ChatGPT | 对话框占满全屏，所有结果在聊天流中展示 | AI 原生场景 | 对话即一切，但传统操作效率低（比如批量编辑 50 条隐患） |
| **侧边栏辅助式** | Microsoft Copilot, Notion AI | 传统界面为主，AI 聊天面板在右侧 | 传统软件加 AI | 不破坏原有工作流，但 AI 感弱，像"附加功能" |
| **双视图融合式** | Cursor, Linear | 传统视图 + AI 视图共存，两者共享上下文，可互相触发 | 需要同时高效操作和 AI 辅助的人 | 最灵活，但设计复杂度高 |

### 1.2 我们应该选哪种？

**选「双视图融合式」**，理由：

1. 安全主任 60% 的时间在做传统操作（查看台账、浏览表格、核对证书），这些用纯聊天效率极低
2. 但安全隐患识别、通知草拟、报告生成这些 AI 场景，又必须用对话
3. 两种操作经常交替——比如在台账里看到一条隐患，想直接让 AI 帮我生成整改通知

**关键设计决策：不是"要么聊天，要么表格"，而是"我随时可以跟 AI 说话，不管我现在在哪个页面"。**

### 2.2 三个区域的职责

| 区域 | 宽度 | 职责 | 什么时候用 |
|------|------|------|-----------|
| **NAV（左侧导航）** | 56px（图标）或 200px（展开） | 切换模块、全局状态指示 | 随时可点 |
| **WORKSPACE（主工作区）** | 弹性，占剩余空间 | 传统 CRUD 操作、数据浏览、可视化 | 浏览/编辑/查看时 |
| **COPILOT PANEL（右侧面板）** | 380px，可收起 | AI 对话、卡片操作、Agent 执行 | 需要 AI 帮助时 |

### 2.3 核心原则：AI 是常驻助手，不是独立功能

**设计铁律：**

> 用户从任何页面都能直接跟 AI 说话，不需要先切到"聊天页面"。AI 的回答与当前工作区上下文联动。

具体实现：

1. **Copilot Panel 始终可见**：默认展开在右侧，点击按钮可收起
2. **Workspace 切换时，Copilot 上下文自动跟随**：比如用户切到隐患台账，Copilot 自动感知"你现在在看 B2 区的隐患"
3. **Workspace 中的操作可以触发 Copilot**：比如在隐患台账选中一条记录，点击"生成整改通知"按钮 → Copilot 自动开始生成
4. **Copilot 的结果可以推送到 Workspace**：比如 AI 生成了一条隐患记录 → 自动出现在隐患台账列表中

**对话视图 vs Copilot Panel 的关系：**

- **对话视图**：完整的历史聊天记录，像 ChatGPT 一样，所有卡片和对话都在这里
- **Copilot Panel**：当用户切到其他视图时，对话视图的内容"缩小"到右侧面板继续

**一句话：对话视图是 Copilot Panel 的全屏版，切走就缩小，切回来就展开。**

### 4.2 上下文联动机制（最关键的设计）

**核心问题：AI 怎么知道用户现在在干什么？**

**解法：自动注入「页面上下文」到 Copilot 对话**

```typescript
// 前端每次切换视图或选中数据时，自动更新 context
interface CopilotContext {
  currentView: 'chat' | 'inspection' | 'hazards' | 'forms' | 'dashboard' | 'risk-radar';
  selectedItems?: {
    type: 'hazard' | 'template' | 'camera' | 'contractor';
    id: string;
    summary: string;  // 一句话描述，如 "B2区临边护栏缺失(高风险)"

  };

}
```

### 8.3 API 调用与 SSE 流式响应

```typescript
// composables/useCopilotChat.ts
async function sendMessage(message: string, attachments?: File[]) {
  const response = await fetch('/api/chat/message', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      message,
      attachments: attachments?.map(f => f.name),
      context: copilotStore.context
    })
  });

  // SSE 流式读取
  const reader = response.body!.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const events = parseSSE(chunk);

    for (const event of events) {
      if (event.type === 'text') {
        // 追加文本到当前消息气泡
        appendToCurrentMessage(event.content);
      } else if (event.type === 'card') {
        // 渲染卡片组件
        addCardMessage(event.card_type, event.data);
      } else if (event.type === 'workspace_update') {
        // 更新 Workspace 数据（如隐患台账新增一条）
        workspaceStore.applyUpdate(event.update);
      }
    }
  }
}
```

## 九、与参考产品的对比总结

| 设计决策 | 我们的方案 | 参考 | 为什么这样做 |
|---------|----------|------|-----------|
| 整体布局 | 三栏：Nav + Workspace + Copilot | Cursor: 编辑器 + Chat Panel | 安全主任需要同时看数据和跟 AI 说话 |
| 对话位置 | 右侧面板（Copilot Panel） | Microsoft Copilot: 右侧栏 | 不遮挡主工作区，随时可用 |
| 对话视图 | Workspace 全屏展开对话 | ChatGPT: 全屏对话 | 纯聊天时给足空间 |
| 视图切换 | 对话 = Copilot 全屏版 | 无直接参考 | 统一体验，切走即缩小，切回即展开 |
| AI 触发点 | Workspace 按钮行内 + Copilot | Linear: Command+K + 行内按钮 | 老手用按钮，新手用对话 |
| 上下文联动 | 自动注入页面上下文 | Cursor: 选中代码自动给 Chat | 用户不需要重复描述"我在看什么" |
| 卡片交互 | 内联编辑 + 操作按钮 | Manus: 对话流内操作 | 操作结果立即可见，不跳转 |
| 确认机制 | 所有执行操作必须点按钮确认 | 原方案 + Microsoft Copilot 原则 | 人在环路，安全第一 |
| 命令面板 | Ctrl+K 全局搜索 | Linear: Command Menu | 快捷操作入口，发现功能 |
