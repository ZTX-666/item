# 赤瞳安全智能平台 — 工具交互架构设计

> 本文档解决核心问题：**5 个已有工具（AgentToolbox / VLM-Detection / WhatsApp Archive / Safety Templates / Report Generators）之间如何交互？数据怎么流？用户怎么用？前端怎么编排？**

---

## 一、现状：5 个工具的定位和当前调用关系

### 1.1 工具定位速查

| 工具 | 语言 | 类型 | 对外接口 | 一句话定位 |
|------|------|------|---------|-----------|
| **AgentToolbox** | Python / FastAPI | HTTP 网关 + MCP 适配器 | `:8899` HTTP + MCP stdio | 统一编排层，封装所有工具为 Agent 可调用 |
| **VLM-Detection** | Python / ultralytics | CLI 推理脚本 | 无（subprocess） | 双 YOLO 检测：工人/PPE + 施工机械 |
| **WhatsApp Archive** | Node.js / Express | HTTP API + 浏览器 UI | `:8787` HTTP | WhatsApp 聊天/消息/附件搜索和下载 |
| **Safety Templates** | 纯数据 | 159 个模板 + JSON 索引 | 无（文件读取） | 安全制度表格模板库 + 制度全文 RAG |
| **Report Generators** | Python / python-docx | 函数库 | 无（importlib） | Word 文档生成器 |

### 1.2 当前调用关系（Hub-and-Spoke）

```
                    Agent 客户端
                   (Cursor/Feishu/MCP)
                         │
                         ▼
               ┌─── AgentToolbox ───┐
               │  HTTP :8899 / MCP   │
               │                     │
    ┌──────────┼──────────┼──────────┼──────────┐
    │          │          │          │          │
    ▼          ▼          ▼          ▼          ▼
 VLM-Detect  WhatsApp  Templates  Reports   External
 (subprocess) (HTTP)   (file read) (import)  (HTTP)
```

**问题：所有交互都要经过 AgentToolbox 这个中心节点。** 这在架构上是干净的（单一入口、统一鉴权审计），但在用户交互层面有两个痛点：

1. **多工具编排的时序对用户不可见** —— 用户说"帮我检查 B 区"，系统要调用 3 个工具（rtmp_snapshot → vlm_detect → ingest_vlm_hazards），用户只看到转圈，不知道在干什么
2. **工具间的数据依赖靠硬编码** —— 比如巡检流程 rtmp_snapshot 的输出路径必须手动传给 vlm_detect，如果其中一个工具格式变了，链路就断了

## 二、交互设计核心原则

### 原则 1：工具编排对用户透明但不打扰

用户不需要知道背后调了几个工具，但**当多工具串联执行时，前端必须展示进度链**：

```
┌─────────────────────────────────────────────────────┐
│  赤瞳 Copilot                                       │
│                                                     │
│  你: 检查一下 B 区摄像头                               │
│                                                     │
│  赤瞳: 正在执行巡检...                                │
│  ┌───────────────────────────────────────────────┐  │
│  │  ● 截取 B 区画面............... 完成 ✓          │  │
│  │  ● VLM 检测识别中.............. 完成 ✓          │  │
│  │  ○ 结果入库待确认.............. 等待中           │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  赤瞳: 检测到 2 个安全隐患 ——                        │
│  [巡检结果卡片]                                      │
└─────────────────────────────────────────────────────┘
```

### 原则 2：工具之间用事件驱动，不用硬编码调用链

当前巡检流程是 `rtmp_snapshot → vlm_detect → ingest_vlm_hazards` 硬编码在 agent-toolbox 里。改进为**事件总线模式**：每个工具完成自己任务后发布事件，下游工具订阅事件自动触发。

### 原则 3：用户可以在任意环节介入

多工具编排不应该是"一键跑完"——用户必须在关键节点确认（入库、发送、生成），这个在改进方案里已经定义了，交互设计要对应实现。

## 四、编排调度层设计（核心新增）

### 4.1 为什么要加编排调度层

当前 AgentToolbox 是"工具路由器"——用户说什么，调什么工具。但真实场景经常需要**多工具串联/并联**：

| 场景 | 需要调用的工具 | 执行顺序 |
|------|--------------|---------|
| 巡检并入库 | rtmp_snapshot → vlm_detect → ingest_vlm_hazards | 串行 |
| 填表并生成 | template_search → safety_form_fill → docx_generate | 串行 |
| 每日简报 | hko_weather + accident_news_search + safety_db_query → LLM 汇总 | 并行 → 汇总 |
| 录入隐患并通知 | ai_archive_classifier → connect_hazard_actions → whatsapp_send | 串行（需人工确认节点） |

### 4.2 Workflow Engine 设计

**不用引入 Airflow / Temporal 这种重量级引擎**——进程内 Python 异步编排即可。

```python
# backend/services/workflow_engine.py

from dataclasses import dataclass, field
from enum import Enum
from typing import Any, Callable, Awaitable
import asyncio
import uuid

class StepStatus(Enum):
    PENDING = "pending"
    RUNNING = "running"
    WAITING_CONFIRM = "waiting_confirm"   # 等人工确认
    COMPLETED = "completed"
    FAILED = "failed"
    SKIPPED = "skipped"

@dataclass
class WorkflowStep:
    """工作流单步"""
    id: str
    name: str                    # 展示给用户的名称，如"截取 B 区画面"
    tool_name: str               # 对应 AgentToolbox 的工具名
    params: dict                 # 工具参数（可引用前步结果）
    depends_on: list[str]        # 依赖的前步 ID 列表
    requires_confirm: bool = False  # 是否需要人工确认
    status: StepStatus = StepStatus.PENDING
    result: Any = None

@dataclass
class Workflow:
    """一次完整的工作流执行"""
    id: str
    name: str                    # "视觉巡检" / "智能填表"
    steps: list[WorkflowStep]
    current_step_index: int = 0
    status: str = "running"
    context: dict = field(default_factory=dict)  # 跨步骤共享数据

class WorkflowEngine:
    def __init__(self, tool_executor, event_emitter):
        self.tool_executor = tool_executor    # 调用 AgentToolbox 的工具
        self.event_emitter = event_emitter    # 向前端推送 SSE 事件
        self.active_workflows: dict[str, Workflow] = {}

    async def start(self, workflow: Workflow) -> str:
        """启动工作流，返回 workflow ID"""
        self.active_workflows[workflow.id] = workflow
        asyncio.create_task(self._run(workflow))
        return workflow.id

    async def _run(self, workflow: Workflow):
        """执行工作流"""
        for step in workflow.steps:
            # 等待依赖完成
            while True:
                deps = [s for s in workflow.steps if s.id in step.depends_on]
                if all(d.status == StepStatus.COMPLETED for d in deps):
                    break
                await asyncio.sleep(0.1)

            # 解析参数（支持引用前步结果）
            resolved_params = self._resolve_params(step, workflow)

            # 更新状态
            step.status = StepStatus.RUNNING
            await self.event_emitter("step_update", {
                "workflow_id": workflow.id,
                "step_id": step.id,
                "status": "running",
                "name": step.name
            })

            # 执行工具
            try:
                result = await self.tool_executor(step.tool_name, resolved_params)
                step.result = result

                if step.requires_confirm:
                    step.status = StepStatus.WAITING_CONFIRM
                    await self.event_emitter("step_waiting", {
                        "workflow_id": workflow.id,
                        "step_id": step.id,
                        "status": "waiting_confirm",
                        "name": step.name,
                        "card": result.get("card")   # 确认卡片
                    })
                    # 暂停，等前端确认回调
                    return

                step.status = StepStatus.COMPLETED
                await self.event_emitter("step_update", {
                    "workflow_id": workflow.id,
                    "step_id": step.id,
                    "status": "completed",
                    "name": step.name
                })

            except Exception as e:
                step.status = StepStatus.FAILED
                await self.event_emitter("step_error", {
                    "workflow_id": workflow.id,
                    "step_id": step.id,
                    "status": "failed",
                    "error": str(e)
                })
                break

    async def confirm_step(self, workflow_id: str, step_id: str, action: str, modifications: dict = None):
        """前端确认某步骤后调用"""
        workflow = self.active_workflows[workflow_id]
        step = next(s for s in workflow.steps if s.id == step_id)

        if action == "confirm":
            step.status = StepStatus.COMPLETED
            # 继续执行后续步骤
            asyncio.create_task(self._run(workflow))
        elif action == "edit":
            # 应用修改后继续
            if modifications:
                step.result = {**step.result, **modifications}
            step.status = StepStatus.COMPLETED
            asyncio.create_task(self._run(workflow))
        elif action == "skip":
            step.status = StepStatus.SKIPPED
            asyncio.create_task(self._run(workflow))

    def _resolve_params(self, step: WorkflowStep, workflow: Workflow) -> dict:
        """解析参数中的引用，如 {{step1.result.output_path}}"""
        import re
        params = step.params.copy()
        for key, value in params.items():
            if isinstance(value, str) and "{{" in value:
                ref_match = re.match(r'\{\{(\w+)\.result\.(\w+)\}\}', value)
                if ref_match:
                    ref_step_id, ref_field = ref_match.groups()
                    ref_step = next(s for s in workflow.steps if s.id == ref_step_id)
                    params[key] = ref_step.result.get(ref_field)
        return params
```

```python
# backend/services/scheduler.py

class InspectionScheduler:
    def __init__(self, workflow_engine, db):
        self.workflow_engine = workflow_engine
        self.db = db
        self.timers: dict[str, asyncio.Task] = {}

    async def start_periodic(self, camera_id: str, interval_minutes: int):
        """启动定时巡检"""
        async def _loop():
            while True:
                camera = self.db.get_camera(camera_id)
                workflow = create_inspection_workflow(camera.area, camera.rtmp_url)
                await self.workflow_engine.start(workflow)
                # 注意：定时巡检的结果不弹卡片，而是写入"待审核队列"
                # 用户在看板或巡检页面可以看到待审核结果
                await asyncio.sleep(interval_minutes * 60)

        self.timers[camera_id] = asyncio.create_task(_loop())

    async def stop_periodic(self, camera_id: str):
        if camera_id in self.timers:
            self.timers[camera_id].cancel()
```

### 场景 3：智能填表

**触发方式**：用户说"帮我填 T006" / 在表格中心点击模板卡片

```python
# backend/tools/template_tools.py

async def safety_form_fill(template_id: str, area: str = None, context_from_db: bool = True):
    """
    智能填表核心逻辑：
    1. 从 table_index.json 加载模板结构
    2. 从 safety_cases 拉取区域相关隐患
    3. 从 WhatsApp 搜索最近相关消息
    4. 从 hko_weather 获取天气数据
    5. 组装预填字段 + 标注来源
    """
    # 加载模板
    template = load_template_json(template_id)   # 从 tables-json/ 读取
    fields = extract_fillable_fields(template)    # 提取可填充字段

    filled = {}
    sources = {}

    if context_from_db:
        # 从 safety_cases 拉相关数据
        cases = query_safety_cases(area=area, status="open")
        for field in fields:
            value = match_field_to_case(field, cases)
            if value:
                filled[field.name] = value
                sources[field.name] = "safety_cases"

        # 从 WhatsApp 搜索相关消息
        recent_msgs = search_whatsapp_messages(query=area)
        for field in fields:
            if field.name not in filled:
                value = extract_from_messages(field, recent_msgs)
                if value:
                    filled[field.name] = value
                    sources[field.name] = "whatsapp"

        # 天气数据
        weather = fetch_hko_weather()
        if "天气" in field.name or "temperature" in field.name.lower():
            filled[field.name] = weather.get("warning_text", "")
            sources[field.name] = "hko_weather"

    return {
        "template_id": template_id,
        "filled_fields": filled,
        "sources": sources,           # 每个字段的数据来源
        "unfilled_fields": [f.name for f in fields if f.name not in filled],
        "card_type": "form_fill"
    }
```

```python
class RiskMonitor:
    """每 30 分钟检测一次外部风险"""

    def __init__(self, workflow_engine, event_emitter, db):
        self.workflow_engine = workflow_engine
        self.event_emitter = event_emitter
        self.db = db
        self.last_weather_warnings = set()
        self.last_news_ids = set()

    async def start(self):
        while True:
            await self._check_weather()
            await self._check_news()
            await asyncio.sleep(30 * 60)

    async def _check_weather(self):
        weather = await self.tool_executor("fetch_hko_weather", {})
        current_warnings = set(w["name"] for w in weather.get("warnings", []))

        # 检测新预警
        new_warnings = current_warnings - self.last_weather_warnings
        if new_warnings:
            # 查数据库中是否有今日进行的高危工序
            active_high_risk = self.db.query_high_risk_activities_today()

            # 推送预警卡片
            await self.event_emitter("risk_alert", {
                "type": "weather",
                "warnings": list(new_warnings),
                "suggested_actions": self._match_actions(new_warnings, active_high_risk),
                "suggested_templates": self._match_templates(new_warnings)
            })

        self.last_weather_warnings = current_warnings

    async def _check_news(self):
        news = await self.tool_executor("fetch_hk_industrial_news", {})
        new_items = [n for n in news if n["id"] not in self.last_news_ids]

        for item in new_items[:3]:   # 最多推送 3 条
            await self.event_emitter("risk_alert", {
                "type": "news",
                "title": item["title"],
                "summary": item["summary"],
                "suggested_actions": self._match_actions_from_news(item),
                "suggested_templates": self._match_templates_from_news(item)
            })

        self.last_news_ids.update(n["id"] for n in news)

    def _match_actions(self, warnings, activities):
        """预警 → 建议行动映射"""
        action_map = {
            "酷热天气警告": [

                "停止露天高处作业",
                "检查临时用电防护",
                "人员撤离到安全区域"
            ]
        }
        actions = []
        for w in warnings:
            actions.extend(action_map.get(w, ["关注天气变化，做好防范"]))
        return actions

    def _match_templates(self, warnings):
        """预警 → 建议表格映射"""

            templates.extend(template_map.get(w, []))
```

### 6.4 前端状态管理（Workflow Store）

```typescript
// frontend/src/stores/workflow.ts

import { defineStore } from 'pinia'

interface WorkflowStep {
  id: string
  name: string
  status: 'pending' | 'running' | 'waiting_confirm' | 'completed' | 'failed'
  result?: any
  card?: any
}

interface Workflow {
  id: string
  name: string
  steps: WorkflowStep[]
  currentStepId?: string
}

export const useWorkflowStore = defineStore('workflow', {
  state: () => ({
    activeWorkflows: new Map<string, Workflow>(),
    // 当前正在等待确认的步骤
    pendingConfirms: [] as Array<{ workflowId: string; stepId: string; card: any }>
  }),

  actions: {
    // SSE 事件处理
    handleStepUpdate(event: { workflow_id: string; step_id: string; status: string; name: string }) {
      const wf = this.activeWorkflows.get(event.workflow_id)
      if (!wf) return
      const step = wf.steps.find(s => s.id === event.step_id)
      if (step) {
        step.status = event.status as any
        step.name = event.name
      }
    },

    handleStepWaiting(event: { workflow_id: string; step_id: string; card: any }) {
      const wf = this.activeWorkflows.get(event.workflow_id)
      if (!wf) return
      const step = wf.steps.find(s => s.id === event.step_id)
      if (step) {
        step.status = 'waiting_confirm'
        step.card = event.card
      }
      this.pendingConfirms.push({
        workflowId: event.workflow_id,
        stepId: event.step_id,
        card: event.card
      })
    },

    async confirmStep(workflowId: string, stepId: string, action: string, modifications?: any) {
      await fetch('/api/workflow/confirm', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ workflow_id: workflowId, step_id: stepId, action, modifications })
      })
      // 从 pendingConfirms 移除
      this.pendingConfirms = this.pendingConfirms.filter(
        p => !(p.workflowId === workflowId && p.stepId === stepId)
      )
    }
  }
})
```

```python
import subprocess
import json

async def whatsapp_send_text(group_name: str, message: str) -> dict:
    """
    通过 whatscli 发送文本消息到 WhatsApp 群组
    whatscli send --group <group_name> --text <message>
    """
    result = subprocess.run(
        ["whatscli", "send", "--group", group_name, "--text", message],

    """
    带确认的发送——必须经人工确认后才实际发送
    这是一个"双阶段工具"：
    - 第一次调用返回草稿（不实际发送）
    - 前端确认后第二次调用（_confirmed=True）才实际发送
    """
    if not _confirmed:
        # 返回草稿卡片，不实际发送
        return {
            "card_type": "notification_draft",
            "data": {
                "target_group": group_name,
                "content_summary": content[:100] + "..." if len(content) > 100 else content,
                "full_content": content,
                "requires_confirm": True
            }
        }
    else:
        # 已确认，实际发送
        return await whatsapp_send_text(group_name, content)

async def whatsapp_groups_list() -> dict:
    """列出可用的 WhatsApp 群组"""
    # 调用 whatsapp-archive API 获取群组列表
    resp = requests.get(f"{WHATSAPP_ARCHIVE_BASE_URL}/api/chats?limit=200")
    chats = resp.json()
    groups = [c for c in chats if c.get("isGroup")]
    return {"groups": [{"jid": g["jid"], "name": g["name"]} for g in groups]}
```
